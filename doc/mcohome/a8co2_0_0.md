# A8-9 Multi-purpose sensor
This describes the Z-Wave device *A8-9*, manufactured by *[McoHome Technology Co., Ltd](http://www.mcohome.com/)* with the thing type UID of ```mcohome_a8co2_00_000```.

The device is in the category of *Sensor*, defining Device used to measure something.

![A8-9 product image](https://opensmarthouse.org/zwavedatabase/1198/image/)


The A8-9 supports routing. This allows the device to communicate using other routing enabled devices as intermediate routers.  This device is also able to participate in the routing of data between other devices in the mesh network.

## Overview

<p&gtMCOHome A8-9 is a Z-Wave enabled multiple environmental monitoring sensors, with 3.5 inch TFT<br /&gtclear display and compliant to Z-Wave Plus standard. It is built in with <strong&gtTemperature, Humidity,</strong&gt<br /&gt<strong&gtPM2.5, CO2, VOC, PIR, illumination, Noise, Smoke</strong&gt sensors. Device can be added into any Z-Wave<br /&gtnetwork, and is compatible with any other Z-Wave certified devices.</p&gt <ul&gt<li&gtTemperature: 0 °C ~ 50 °C</li&gt <li&gtHumidity: 0% RH ~ 99% RH</li&gt <li&gtPM2.5: 0 µg / m<sup&gt3 </sup&gt~ 500 µg / m<sup&gt3</sup&gt</li&gt <li&gtCO<sub&gt2</sub&gt: 0 ppm ~ 5000 ppm</li&gt <li&gtVOC: 0 ppb - 64000 ppb</li&gt <li&gtIllumination: 0 Lux ~ 40000 Lux</li&gt <li&gtNoise: 30 dB ~ 100 dB</li&gt <li&gtPIR: 0 or 1 detection angle up to 120°</li&gt <li&gtSmoke: 0 or 1</li&gt </ul&gt<p&gt<strong&gtSpecification</strong&gt</p&gt <ul&gt<li&gtDimension: 110 \* 110 \* 32 mm</li&gt <li&gtSelf-dissipation: < 3 Watt</li&gt <li&gtHole Pitch: 60 mm or 82 mm</li&gt <li&gtWork environment: -20 °C ~ + 60 °C < 99% RH (non-condensation)</li&gt <li&gtHousing: Tempered glass + PC Alloy</li&gt <li&gtInstallation: Wall-mounted (vertical)</li&gt </ul&gt<p&gt<strong&gtCommand Class supported by the device: (Supports S2 unauthenticated level)</strong&gt</p&gt <ul&gt<li&gtCOMMAND\_CLASS\_VERSION</li&gt <li&gtCOMMAND\_CLASS\_MANUFACTURER\_SPECIFIC</li&gt <li&gtCOMMAND\_CLASS\_DEVICE\_RESET\_LOCALLY</li&gt <li&gtCOMMAND\_CLASS\_POWERLEVEL</li&gt <li&gtCOMMAND\_CLASS\_ASSOCIATION</li&gt <li&gtCOMMAND\_CLASS\_ASSOCIATION\_GRP\_INFO</li&gt <li&gtCOMMAND\_CLASS\_CONFIGURATION</li&gt <li&gtCOMMAND\_CLASS\_SENSOR\_MULTILEVEL</li&gt <li&gtCOMMAND\_CLASS\_FIRMWARE\_UPDATE\_MD</li&gt </ul&gt<p&gt<strong&gtCommand Class supported by the device: (Not supporting S2)</strong&gt</p&gt <ul&gt<li&gtCOMMAND\_CLASS\_ZWAVEPLUS\_INFO</li&gt <li&gtCOMMAND\_CLASS\_TRANSPORT\_SERVICE\_V2</li&gt <li&gtCOMMAND\_CLASS\_SECURITY\_2</li&gt <li&gtCOMMAND\_CLASS\_SUPERVISION</li&gt </ul&gt<p&gt<strong&gtNote: A Security Enabled Z-Wave Controller must be used in order to fully utilise the product</strong&gt</p&gt

### Inclusion Information

<p&gt<strong&gtAdd to Z-Wave network</strong&gt</p&gt <ul&gt<li&gtActivate inclusion mode in the gateway. When device is powered on, hold F1 to select the interface for Z-Wave network inclusion.</li&gt <li&gtClick F2 five times until <em&gt<refresh icon></em&gt turns blue.</li&gt <li&gtHold F2 and the device enters into learning mode, then <em&gt<network icon></em&gt turns blue and the device is added into Z-Wave network.</li&gt </ul&gt

### Exclusion Information

<p&gt<strong&gtRemove from Z-Wave network</strong&gt</p&gt <ul&gt<li&gtActivate exclusion mode in the gateway. When device is powered on, hold F1 to select the interface for Z-Wave network exclusion.</li&gt <li&gtClick F2 five times until <em&gt<refresh icon></em&gt turns blue.</li&gt <li&gtHold F2 and the device enters into learning mode, then <em&gt<network icon></em&gt turns blue and the device is added into Z-Wave network.</li&gt </ul&gt

### General Usage Information

<p&gt<strong&gtPower On / Power Off</strong&gt</p&gt <p&gtWire the adapter and the device is powered on. It will display all detected information by the sensors.</p&gt <p&gt<strong&gtDisplay Interface</strong&gt</p&gt <p&gtHold Key F1 can switch among the following 4 display interfaces:</p&gt <ol&gt<li&gtData detecting: display all sensors’ data</li&gt <li&gtNetwork: Z-Wave Add/Remove</li&gt <li&gtData calibration: to calibrate the detected data manually</li&gt <li&gtLocal time setting</li&gt </ol&gt<p&gt<strong&gtRestore Factory Setting</strong&gt</p&gt <ol&gt<li&gtPress & hold F1 to enter Z-Wave setting interface, then press & hold F1 again to enter parameters<br /&gtsetting interface;</li&gt <li&gtPress & hold F2 to enter setting interface and select “default”;</li&gt <li&gtClick F2 3 times and displays “OFF”-->“ON”-->“OK”-->“OFF”, factory setting is restored.</li&gt </ol&gt<p&gt<em&gtNote: Please use this procedure only when the network primary controller is missing or otherwise inoperable.</em&gt</p&gt <p&gt<strong&gtData Calibration</strong&gt</p&gt <p&gtHold F1 to choose interface for data calibration. Then hold F2 to switch among the sensors. Choose one and click F2, F1 to change the data. After finished, hold F1 can return data detecting interface.</p&gt <p&gt<strong&gtLocal Time Setting</strong&gt</p&gt <p&gtHold F1 to choose interface for local time setting. Then hold F2 to switch among “Hour-Minute-Second-Year-Month-Date”. Click F2, F1 can change the data of flashing item. After finished, hold F1 can return data detecting interface.</p&gt

## Association Groups

Association groups allow the device to send unsolicited reports to the controller, or other devices in the network. Using association groups can allow you to eliminate polling, providing instant feedback of a device state change without unnecessary network traffic.

The A8-9 supports 1 association group.

### Group 1: Lifeline

The Lifeline association group reports device status to a hub and is not designed to control other devices directly. When using the Lineline group with a hub, in most cases, only the lifeline group will need to be configured and normally the hub will perform this automatically during the device initialisation.

Association group 1 supports 1 node.
